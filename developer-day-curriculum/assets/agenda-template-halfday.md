# MongoDB Workshop - Half-Day Format

**Date:** [Date]  
**Duration:** 3.5 hours  
**Level:** Intermediate

## Learning Objectives

- Understand MongoDB document model and when to use it
- Design effective schemas for common use cases
- Write aggregation pipelines for data analysis

## Prerequisites

- Basic MongoDB knowledge (or completed "Intro to MongoDB" course)
- Laptop with MongoDB installed or Atlas account
- Node.js or Python installed

## Schedule

### 9:00 AM - Welcome & Introductions (15 min)

- Icebreaker activity
- Learning objectives
- Wi-Fi and logistics

### 9:15 AM - MongoDB Fundamentals (30 min)

**Learning objective:** Refresh on MongoDB basics

- Document model overview
- CRUD operations quick review
- Comparison with relational databases

### 9:45 AM - Exercise 1: Your First MongoDB Application (30 min)

**Goal:** Build a basic CRUD application

- Starter code: `exercises/01-crud/`
- Insert, find, update, delete documents
- Success criteria: All tests passing

### 10:15 AM - Break (15 min)

### 10:30 AM - Schema Design Patterns (45 min)

**Learning objective:** Design effective MongoDB schemas

- Embedding vs referencing
- One-to-many relationships
- Schema design anti-patterns
- Live demo: E-commerce schema

### 11:15 AM - Exercise 2: Design Your Schema (60 min)

**Goal:** Design and implement a schema for [your use case]

- Starter code: `exercises/02-schema-design/`
- Model entities and relationships
- Validate with sample queries
- Extension: Add indexes

### 12:15 PM - Wrap-Up & Q&A (15 min)

- Review learning objectives
- Next steps and resources
- Feedback survey
