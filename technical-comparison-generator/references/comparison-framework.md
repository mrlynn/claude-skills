# Comparison Framework

## Golden Rule

**Help developers choose the right tool, even if it's not MongoDB.**

## Principles

### 1. Be Fair
- Acknowledge competitor strengths
- Use current versions
- Don't cherry-pick features
- Cite sources

### 2. Focus on Use Case
- Not "better/worse" - "fits better for X"
- Specific requirements matter
- Context is everything

### 3. Be Specific
- Vague: "MongoDB is faster"
- Specific: "MongoDB horizontal scaling handles 100k writes/sec on commodity hardware"

## Structure

1. **Competitor strengths** (acknowledge first)
2. **MongoDB strengths** (honest assessment)
3. **Use case fit** (when to use each)
4. **Migration path** (if switching)

## Quality Checks

- [ ] Fair to competitor
- [ ] Specific (not vague)
- [ ] Use case focused
- [ ] Sources cited
- [ ] Current versions
