# Talk Structures

## 1. Problem-Solution-Demo

**Best for:** Technical deep dives

**Structure:**
- Problem (5 min)
- Solution (20 min)
- Demo (15 min)
- Q&A (5 min)

## 2. Story-Lesson-Application

**Best for:** Experience reports

**Structure:**
- Story (10 min)
- Extract lessons (10 min)
- Apply to audience (20 min)
- Q&A (5 min)

## 3. Journey (Progressive)

**Best for:** Tutorials

**Structure:**
- Start simple (5 min)
- Add complexity (15 min)
- Handle edge cases (15 min)
- Production tips (10 min)

## 4. Pattern Showcase

**Best for:** Best practices

**Structure:**
- Overview (5 min)
- Pattern 1 (10 min)
- Pattern 2 (10 min)
- Pattern 3 (10 min)
- When to use each (5 min)
- Q&A (5 min)

## Timing Rules

- **1 slide/minute** for technical talks
- **2-3 min/slide** for code-heavy slides
- **Always buffer 5-10 min** for overruns

## Demo Best Practices

- Rehearse 5+ times
- Record backup video
- Have fallback slides
- Test on conference WiFi
