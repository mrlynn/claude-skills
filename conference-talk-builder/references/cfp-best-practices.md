# CFP Best Practices

## The Formula

**Hook + Problem + Solution + Proof = Accepted CFP**

## Hook (First 2 Sentences)

Grab attention immediately.

**Bad:** "This talk will cover MongoDB features."
**Good:** "Your queries are slow. I'll show you the 3 patterns that cut response time by 80%."

## Problem Statement

Be specific about the pain.

**Vague:** "Developers struggle with databases."
**Specific:** "Migrating from SQL to MongoDB, developers waste weeks debating embed vs reference decisions."

## Solution

Concrete, actionable approach.

**Vague:** "We'll discuss best practices."
**Specific:** "6-factor decision framework tested on 100+ production schemas."

## Learning Objectives

3-5 specific takeaways.

**Format:** "Attendees will learn to [ACTION] [OUTCOME]"

**Example:**
- Build a RAG system handling 10M documents with <200ms latency
- Implement incremental ingestion (90% cost savings)
- Monitor retrieval quality with MRR and Precision@K

## Target Audience

Be specific.

**Vague:** "Developers"
**Specific:** "Backend developers building AI applications with Python"

## Proof of Expertise

Why should they listen to you?

**Generic:** "I work at MongoDB."
**Compelling:** "Built RAG systems for 5 Fortune 500 companies processing 50M+ embeddings."

## Length

- **Sweet spot:** 250-400 words
- **Too short:** <200 (not enough detail)
- **Too long:** >500 (reviewers skim)

## Common Mistakes

❌ No hook
❌ Vague promises ("we'll discuss...")
❌ Missing learning objectives
❌ No proof of expertise
❌ Typos
